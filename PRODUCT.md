# Product

## Register

product

## Users

Developers checking their local machine's resource usage during development, debugging, or builds. They glance at the dashboard to answer: "Is my build eating all the RAM?" "Is the GPU still training?" "What's hogging the CPU?"

## Product Purpose

Real-time system load dashboard that fetches from the System Load API and displays CPU, memory, disk, GPU, and network metrics at a glance. Auto-refreshes every 5 seconds. Single-page, no auth, no history.

## Brand Personality

Terminal-confident. Dense but scannable. No decoration that doesn't serve data. Feels like a tool you'd find in a well-curated developer toolkit.

## Anti-references

- Grafana/Datadog (too heavy, too many panels)
- Generic SaaS dashboards with gradient cards and hero metrics
- Light-themed admin panels with side navs and breadcrumbs
- Anything that looks like it was generated by an AI dashboard template

## Design Principles

1. Data first, chrome second — every pixel serves a metric
2. Terminal-native aesthetic — monospace, dark, high-contrast
3. Scannable at a glance — hierarchy through size and weight, not color alone
4. No wasted space — dense information, no decorative whitespace
5. Graceful degradation — GPU missing, API down, empty states all handled cleanly

## Accessibility & Inclusion

WCAG AA. Color is never the sole indicator of state. Reduced motion support.
